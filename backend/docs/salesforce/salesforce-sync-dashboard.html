🏢 Companies Sync ▶

POST /salesforce/sync/companies Sync all companies to Salesforce
POST /salesforce/sync/company/{id} Sync specific company by ID

👥 Users Sync ▶

🔄 New Flow (Phase 2): Users sync as Leads → CompanyUsers sync as Contacts
POST /salesforce/sync/users Sync all users as Leads
POST /salesforce/sync/user/{id} Sync specific user as Lead by ID
POST /salesforce/convert-lead/{userId} Convert Lead to Account and Contact

🤝 CompanyUsers Sync ▶

✨ CompanyUsers → Contact Sync — syncs user company memberships as Salesforce Contacts
POST /salesforce/sync/company-users 🆕 Sync all CompanyUsers (memberships) to Contacts
POST /salesforce/sync/company-user/{id} 🆕 Sync CompanyUser (membership) to Contact by ID

📋 Listings Sync ▶

POST /salesforce/sync/listing/{id} Sync specific listing by ID
POST /salesforce/sync/wanted-listing/{listingId} Sync specific wanted listing
POST /salesforce/sync/all-wanted-listings Sync All Wanted Listings

💰 Offers Sync ▶

POST /salesforce/sync/offers Sync all offers to Salesforce
POST /salesforce/sync/offers/{offerId} Sync specific offer by ID

🚚 Haulage Offers Sync ▶

POST /salesforce/sync/haulage-offers Sync all haulage offers to Salesforce
POST /salesforce/sync/haulage-offers/{haulageOfferId} Sync specific haulage offer by ID
POST /salesforce/pull/haulage-offer/{haulageOfferId} ⬇️ Pull single haulage offer from Salesforce to WasteTrade

🚚 Haulage Loads Sync ▶

🆕 Phase 2: Load Details Integration - Auto-generated when haulage offer accepted, syncs to Salesforce Haulage_Loads__c
POST /salesforce/sync/haulage-loads Sync all haulage loads to Salesforce
POST /salesforce/sync/haulage-load/{loadId} Sync specific haulage load by ID
GET /haulage-offers/{id}/loads View loads for haulage offer
⬇️ Pull from Salesforce (Inbound Sync)
POST /salesforce/pull/haulage-loads/{haulageOfferId} 🆕 Pull load updates from Salesforce (weight, collection date, status)
Auto-Generation: Loads are automatically created when haulage offer is accepted
Sync Flow: WasteTrade → Salesforce (outbound), Salesforce → WasteTrade (inbound webhook)
Auto-Update: When SF updates load status to "Delivered", offer's shippedLoads count is auto-updated
Setup Required: See docs/salesforce/apex/HaulageLoadsTrigger.trigger for Apex trigger setup

⬇️ Inbound Sync (SF → WasteTrade) ▶

🔄 Pull Updates from Salesforce

Manually trigger inbound sync to pull changes from Salesforce to WasteTrade

POST /salesforce/pull-updates ⬇️ Pull all updates from Salesforce (Accounts, Contacts, Haulage Offers)
What gets synced:
  • Accounts: Company name, VAT, registration number, address
  • Contacts: User name, phone, role, status
  • Haulage Offers: Transit time, transport provider, container type, demurrage, collection date, customs clearance
Note: Only records modified in SF after last sync will be updated (default: last 60 minutes)
