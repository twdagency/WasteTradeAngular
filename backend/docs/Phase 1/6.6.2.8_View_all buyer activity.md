# View all buyer activity === View Purchases & Filter (Admin) 
It same 6.6.1.4 but here have some modification need to configure in both task.
Filter now will be input & not multiple choice so now it will be string for each field.
Admin-only endpoint to retrieve and filter all Purchases in the system.
Endpoint: `GET /offers/admin`
## Query Parameters:
filter: JSON object containing filtering options
### skip: Number of records to skip (pagination)
### limit: Maximum number of records to return.
### where: Filtering conditions:
#### buyerCompanyName: String value to filter by buyer company name.
#### sellerCompanyName: String value to filter by seller company name.
#### searchTerm: Text to search across multiple fields like buyer company name, seller company name, material type, material packing, location.
#### materialType:  of material types to filter by.
#### materialPacking:  of material packing types to filter by.
#### location:  of locations (countries) to filter by.
#### status:  of offer statuses to filter by.
#### state:  of offer states to filter by.

## Default if selected 'All', don't need to include field in `where`. Because of lack of information about order by, already set default order by new purchases.

## Example Filter query
{
    "skip": 0,
    "limit": 20,
    "where": {
        "buyerCompanyName": "Waste Trade",
        "buyerName":"ABC",
        "sellerCompanyName": "Waste Trade",
        "sellerName":"DEF",
        "searchTerm":"something",
        "materialType": "plastic",
        "materialPacking":"something",
        "location":"United Kingdom",
        "status":"pending",
        "state":"active"
    }
}
