## cURL Wanted Listing

This API to get Listing wanted Section Including information company (currently don't have filter, just pagination)

```bash
curl -L -X GET 'https://wastetrade-api-dev.b13devops.com/listings/wanted?filter=%7B%0A%20%20%22limit%22%3A%2020%2C%0A%20%20%22skip%22%3A%200%0A%7D' \
-H 'accept: application/json' \
-H 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6NSwiY3JlYXRlZEF0IjoiMjAyNS0wNS0wMlQwMDoxNjoxMS4wNzhaIiwiaWF0IjoxNzQ2MTQ0OTcxLCJleHAiOjM0OTIyOTk5NDJ9.3oRxDMNGnQZ6rL7iwfOpIrMF-r7V0cARmbM2HVRwWZU'
```

## Filter Parameter

The filter parameter is a **JSON object** that can include the following fields ase on design:

### 1. **`skip`** (integer)

Defines the number of records to skip for pagination.  
**Default**: 0

### 2. **`limit`** (integer)

Defines the number of records to return per page.  
**Default**: 20

### 3. **`where`** (object)

The `where` object contains the filtering conditions applied to the listings.

- `dateRequiredFrom`: timestamp defined.

- `dateRequiredTo`: timestamp defined.

- `company`: company name of buyer (string).

- `listingType`: Should be set to `"wanted"` as the only valid option for listing wanted material.

- `materialType`: String value to filter the listings by the material type (e.g., `"plastic"`). If user chooses `"All"` option, just ignore this field in `filter`

- `materialPacking`: String value to filter the listings by material packing type (e.g., `"bags"`). If user chooses `"All"` option, just ignore this field in `filter`

- `country`: String value to filter the listings by country (e.g., `"USA"`). If user chooses `"All"` option, just ignore this field in `filter`

- `status`: String value to filter the listings by status (e.g, `"active"`), if user chooses `"All"`, just ignore this field in `filter`

- `state`: String value to filter the listings by state (e.g, `"active"`), if user chooses `"All"`, just ignore this field in `filter`

- `sortBy`: ListingSorbyEnum. In default will be created_at desc

```bash
export enum ListingSortBy {
    COMPANY_NAME_ASC = 'companyNameAsc',
    COMPANY_NAME_DESC = 'companyNameDesc',
    MATERIAL_TYPE_ASC = 'materialTypeAsc',
    MATERIAL_TYPE_DESC = 'materialTypeDesc',
    COUNTRY_ASC = 'countryAsc',
    COUNTRY_DESC = 'countryDesc',
}
```

**Default**: 0

### 2. **`limit`** (integer)

Defines the number of records to return per page.  
**Default**: 20

### 3. **`where`** (object)

The `where` object contains the filtering conditions applied to the listings.

- `searchTerm`: When user input in search bar, it filter materialType or materialPacking or materialItem. Otherwise, just ignore this field in `filter`
- `buyerCompanyName`: String value to filter by buyer company name (e.g., "Waste Trade"). Otherwise, just ignore this field in `filter`
- `sellerCompanyName`: String value to filter by seller company name (e.g., "Waste Trade"). Otherwise, just ignore this field in `filter`
- `materialType`: An array of values to filter the listings by the material type (e.g., `["plastic", "metal"]`). Base on design, maybe it can hold multiple options so materialType defined in array. If user choose `"All"` option, just ignore this field in `filter`

- `materialItem`: An array of values to filter the listings by material item (e.g., `["item1", "item2"]`). Base on design, maybe it can hold multiple options so materialItem defined in array. If user choose `"All"` option, just ignore this field in `filter`

- `materialPacking`: An array of values to filter the listings by material packing type (e.g., `["bags", "bales"]`). Base on design, maybe it can hold multiple options so materialPacking defined in array. If user choose `"All"` option, just ignore this field in `filter`
- `materialPacking`: An array of values to filter the listings by material packing type (e.g., `["bags", "bales"]`). Base on design, maybe it can hold multiple options so materialPacking defined in array. If user choose `"All"` option, just ignore this field in `filter`
- `country`: An array of countries to filter the listings (e.g., `["USA", "Germany"]`). Base on design, maybe it can hold multiple options so country defined in array. If user choose `"All"` option, just ignore this field in `filter`

- `status`: An array of status to filter the listings (e.g, `["active, pending]`), if user choose `"All"`, just ignore this field in `filter`

- `state`: An array of state to filter the listings (e.g, `["active, pending]`), if user choose `"All"`, just ignore this field in `filter`

- `sortBy`: It can be `"available"` or `"unavailable"`. You need to define this field in `filter`
