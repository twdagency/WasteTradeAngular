## Filter Parameter
The filter parameter is a **JSON object** that can include the following fields ase on design:
### 1. **`skip`** (integer)  
Defines the number of records to skip for pagination.  
**Default**: 0
### 2. **`limit`** (integer)  
Defines the number of records to return per page.  
**Default**: 20
### 3. **`where`** (object)  
The `where` object contains the filtering conditions applied to the listings.  

- `company`: company name of seller.

- `listingType`: Should be set to `"sell"` as the only valid option for listing sell material.

- `materialType`: An array of values to filter the listings by the material type (e.g., `["plastic", "metal"]`). Base on design, maybe it can hold multiple options so materialType defined in array. If user choose `"All"` option, just ignore this field in `filter`

- `country`: An array of countries to filter the listings (e.g., `["USA", "Germany"]`). Base on design, maybe it can hold multiple options so country defined in array. If user choose `"All"` option, just ignore this field in `filter`

- `status`: An array of status to filter the listings (e.g, `["active, pending]`), if user choose `"All"`, just ignore this field in `filter`

- `state`: An array of state to filter the listings (e.g, `["active, pending]`), if user choose `"All"`, just ignore this field in `filter`

- `sortBy`: ListingSorbyEnum. In default will be created_at desc

``` bash
export enum ListingSortBy {
```
