# 6.1.1.1 Registration Trading Platform API Documentation

This API handles the initial user registration process for Trading Platform users.

## API Endpoint

```
POST /register-trading
```

Creates a new user account and company for trading platform access.

## Request Body

All fields are required unless marked as optional.

```json
{
  "email": "newuser@example.com",
  "password": "securepassword", 
  "firstName": "John",
  "lastName": "Doe",
  "prefix": "Mr.",
  "jobTitle": "Helpdesk",
  "phoneNumber": "1234567890",
  "mobileNumber": "0987654321",
  "whereDidYouHearAboutUs": "Online",
  "companyName": "Waste Trade",
  "companyInterest": ["Buyer", "Seller"],
  "favoriteMaterials": ["Wood", "Metal"]
}
```

### Field Descriptions

- `email` - User's email address (must be unique)
- `password` - Password for the account (minimum 8 characters)
- `firstName` - User's first name
- `lastName` - User's last name
- `prefix` - Title/prefix (Mr., Mrs., Ms., Dr., etc.)
- `jobTitle` - User's job title
- `phoneNumber` - User's phone number
- `mobileNumber` - User's mobile number
- `whereDidYouHearAboutUs` - Marketing attribution
- `companyName` - Name of the company
- `companyInterest` - Array of interests: "Buyer", "Seller"
- `favoriteMaterials` - Array of preferred material types

## Notes

1. **Access Token**: The returned access token should be used for subsequent API calls requiring authentication
2. **Company Status**: New companies start with `pending_verification` status
3. **User Role**: The registering user becomes the company admin
4. **Next Steps**: After registration, users should complete company information and upload required documents

## Related Endpoints

- `PATCH /companies/{id}` - Update company information
- `POST /company-documents` - Upload company documents
- `POST /company-locations` - Add company locations
- `GET /confirmation` - Send confirmation email
