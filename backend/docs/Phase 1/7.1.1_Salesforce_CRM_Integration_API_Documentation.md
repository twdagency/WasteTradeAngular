# 6.9.1 Salesforce Integration API Documentation

This API provides comprehensive Salesforce CRM integration for the WasteTrade platform, enabling automatic synchronization of companies, users, listings, offers, and documents.

## API Endpoints

### Health Check & Connection

#### Check Salesforce Service Health
```
GET /salesforce/health
```

Basic health check for the Salesforce service.

##### cURL Example
```bash
curl -L -X GET 'https://wastetrade-api-dev.b13devops.com/salesforce/health'
```

##### Response
```json
{
  "status": "ok",
  "timestamp": "2025-01-16T10:00:00.000Z"
}
```

#### Test Salesforce Connection
```

```

Tests the actual connection to Salesforce to verify authentication and access.

##### cURL Example


### Synchronization Endpoints

#### Sync All Data
```
POST /salesforce/sync/all
```

Synchronizes all data types to Salesforce in logical order: companies → users → listings → offers → documents.

##### Query Parameters
- `forceSync` (boolean, optional) - Force sync even if already synced (default: false)
- `limit` (number, optional) - Limit number of records to sync

##### cURL Example
```bash
curl -L -X POST 'https://wastetrade-api-dev.b13devops.com/salesforce/sync/all?forceSync=false&limit=100' \
-H 'Content-Type: application/json'
```

##### Response
```json
{
  "status": "success",
  "message": "All data synchronized successfully",
  "data": {
    "totalRecords": 500,
    "totalSuccessful": 485,
    "totalFailed": 15,

        "skipped": 0

      "users": {
        "total": 200,
        "successful": 195,

        "skipped": 0

        "total": 150,
```

### Document Synchronization

```
POST /salesforce/sync/documents/company
```

Synchronizes all company documents to Salesforce.

##### Query Parameters
- `forceSync` (boolean, optional) - Force sync existing documents

##### cURL Example
```bash
curl -L -X POST 'https://wastetrade-api-dev.b13devops.com/salesforce/sync/documents/company?forceSync=true'
```

##### Response
```json
{
  "status": "success", 
  "message": "Synced 45 documents, 2 failed, 5 skipped",
  "data": {
    "total": 52,
    "successful": 45,
    "failed": 2,
    "skipped": 5
  }
}
```

#### Sync Specific Company Document
```
POST /salesforce/sync/documents/company/{documentId}
```

Synchronizes a specific company document by ID.

##### Path Parameters
- `documentId` (number, required) - ID of the company document to sync

##### Query Parameters
- `forceSync` (boolean, optional) - Force sync even if already synced

##### cURL Example
```bash
curl -L -X POST 'https://wastetrade-api-dev.b13devops.com/salesforce/sync/documents/company/123?forceSync=false'
```

##### Response - Success
```json
{
  "status": "success",
  "message": "Document synced successfully",
  "data": {
    "success": true,
    "salesforceId": "a031234567890ABC",
    "documentId": 123
  }
}
```

##### Response - Failure
```json
{
  "status": "error",
  "message": "Document sync failed",
  "data": {
    "success": false,
    "error": "Invalid document type",
    "documentId": 123
  }
}
```

## Authentication & Configuration

The Salesforce integration requires the following environment variables:

```bash
# Salesforce OAuth2 Configuration
SALESFORCE_CLIENT_ID=your_connected_app_client_id
SALESFORCE_CLIENT_SECRET=your_connected_app_client_secret
SALESFORCE_USERNAME=your_salesforce_username
SALESFORCE_PASSWORD=your_salesforce_password
SALESFORCE_SECURITY_TOKEN=your_security_token

# Salesforce Instance URL
SALESFORCE_LOGIN_URL=https://login.salesforce.com  # or https://test.salesforce.com for sandbox

# Environment Prefix (optional)
SALESFORCE_ENVIRONMENT_PREFIX=DEV_  # Prefixes custom field names
```

## Error Handling

### Common Error Responses

#### 500 - Salesforce Connection Error
```json
{
  "status": "error",
  "message": "Error connecting to Salesforce",
  "data": {
    "error": "INVALID_LOGIN: Invalid username, password, security token; or user locked out."
  }
}
```

#### 500 - Sync Error
```json
{
  "status": "error", 
  "message": "Error syncing data",
  "data": {
    "error": "REQUIRED_FIELD_MISSING: Required fields are missing: [Name]",
    "recordId": 123
  }
}
```

#### 429 - Rate Limit Exceeded
```json
{
  "status": "error",
  "message": "Salesforce API rate limit exceeded",
  "data": {
    "error": "REQUEST_LIMIT_EXCEEDED",
    "retryAfter": 300
  }
}
```

## Sync Status Tracking

Each record in the database has Salesforce-related fields:

- `salesforceId` - The Salesforce record ID
- `salesforceSyncedAt` - Timestamp of last successful sync
- `salesforceSyncStatus` - Status of sync (pending, success, failed)
- `salesforceSyncError` - Error message if sync failed

## Best Practices

### Sync Strategy
1. **Initial Sync**: Run full sync with `forceSync=false` to sync new records only
2. **Incremental Sync**: Schedule regular syncs to catch updates
3. **Force Sync**: Use `forceSync=true` sparingly for troubleshooting
4. **Batch Processing**: Use `limit` parameter for large datasets

### Error Handling
1. **Retry Logic**: Implement retry logic for transient errors
2. **Error Monitoring**: Monitor sync errors and address systematically
3. **Data Validation**: Validate data before sync to prevent errors
4. **Partial Failures**: Handle partial failures gracefully

### Performance
1. **Bulk Operations**: Salesforce APIs support bulk operations for better performance
2. **Rate Limiting**: Respect Salesforce API rate limits
3. **Selective Sync**: Only sync changed records when possible
4. **Monitoring**: Monitor API usage and performance metrics

## Monitoring & Logging

### Sync Monitoring
- All sync operations are logged with timestamps and results

- Sync statistics are available in responses

### Salesforce Limits
- Daily API request limits are monitored
- Bulk API limits are tracked

## Troubleshooting

### Common Issues

#### Authentication Problems
- Verify credentials and security token
- Check IP restrictions in Salesforce
- Ensure Connected App is properly configured

#### Field Mapping Errors
- Verify custom fields exist in Salesforce
- Check field permissions for the integration user
- Validate field data types and lengths

#### Sync Failures
- Review error messages in sync responses
- Check Salesforce debug logs
- Verify data integrity in source system

### Debug Mode
Enable debug mode with environment variable:
```bash
SALESFORCE_DEBUG=true
```

This provides detailed logging of API requests and responses.

## Security Considerations

1. **Credentials**: Store Salesforce credentials securely
2. **Network**: Use HTTPS for all API communications
3. **Access Control**: Limit access to sync endpoints
4. **Audit Trail**: All sync operations are logged for audit purposes
5. **Data Privacy**: Ensure compliance with data protection regulations
