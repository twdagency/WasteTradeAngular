## API GET listing wanted material
 The `GET /listings` endpoint retrieves a list of listings with optional filter

 ## Example cURL 
 ```bash
curl -L -X GET 'http://127.0.0.1:3000/listings?filter=%7B%0A%20%20%22limit%22%3A%2010%2C%0A%20%20%22skip%22%3A%200%2C%0A%20%20%22where%22%3A%20%7B%0A%20%20%20%20%20%20%22listingType%22%3A%22wanted%22%0A%20%20%20%20%7D%0A%7D' \
-H 'accept: application/json'
 ```

## Filter Parameter
The filter parameter is a **JSON object** that can include the following fields ase on design:
### 1. **`skip`** (integer)  
Defines the number of records to skip for pagination.  
**Default**: 0
### 2. **`limit`** (integer)  
Defines the number of records to return per page.  
**Default**: 10
### 3. **`where`** (object)  
The `where` object contains the filtering conditions applied to the listings.  
- `listingType`: Should be set to `"wanted"` as the only valid option for listing wanted material.

- `materialType`: An array of values to filter the listings by the material type (e.g., `["plastic", "metal"]`). Base on design, maybe it can hold multiple options so materialType defined in array. If user choose `"All"` option, just ignore this field in `filter`

- `materialItem`: An array of values to filter the listings by material item (e.g., `["item1", "item2"]`). Base on design, maybe it can hold multiple options so materialItem defined in array. If user choose `"All"` option, just ignore this field in `filter`

- `materialPacking`: An array of values to filter the listings by material packing type (e.g., `["bags", "bales"]`). Base on design, maybe it can hold multiple options so materialPacking defined in array. If user choose `"All"` option, just ignore this field in `filter`

- `country`: An array of countries to filter the listings (e.g., `["USA", "Germany"]`). Base on design, maybe it can hold multiple options so country defined in array. If user choose `"All"` option, just ignore this field in `filter`

- `wasteStoration`: It can be `"indoor"` or `"outdoor"`. If user choose both `"indoor"` and `"outdoor"`. Just ignore this field in `filter`
- `showFullfilledListing`: When user toggle checkbox "show fullfilled listings" it can have value `true`. Otherwise, just ignore this field in `filter`

- `searchTerm`: When user input in search bar, it filter materialType or materialPacking or materialItem. Otherwise, just ignore this field in `filter`
## Example full fields

Filter Object (query)
```bash
{
  "limit": 10,
  "skip": 0,
  "where": {
      "listingType":"wanted",
      "country":["belgium","estonia"]
      "materialType":["plastic","material"],
      "materialItem":["pe_pa"],
      "materialPacking":"["bags"],
      "wasteStoration":"outdoor",
      "showFullfilledListing":true,
      "searchTerm":"efw"
    }
}
```
