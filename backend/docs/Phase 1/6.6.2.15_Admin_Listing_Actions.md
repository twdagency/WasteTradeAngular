# Admin Listing Actions

Admin-only endpoint to perform actions on listings (approve/reject/request information).
Endpoint: `PATCH /listings/admin/{id}/{requestAction}`

## Path Parameters

### id: Listing ID (number)

### requestAction: Action to perform (string)

##### "approve": Approve the listing

##### "reject": Reject the listing

##### "request_information": Request additional information

## Request Body

```json
{
    "rejectionReason": "string (optional, required for reject action, if don't provide, record will have default rejectReason)",
    "message": "string (optional, required for request_information action, if don't provide, record will have default message)"
}
```

```bash
export enum ListingRequestActionEnum {
    ACCEPT = 'accept',
    REJECT = 'reject',
    REQUEST_INFORMATION = 'request_information',
}
```

## Authorization

Requires JWT token with admin or super admin role.

## Actions and Effects

### Approve Action

- Changes listing state to "active"
- No additional request body required

### Reject Action

- Changes listing state to "rejected"
- Sends rejection email with provided reason
- Requires rejectionReason, if not have rejectReason will have default reason `"Unsuitable listing"` in request body

### Request Information Action

- Keeps listing state as "pending"
- Sends email requesting additional information
- Requires message, if not have message will have default message `"Please check the listing and provide more information"` in request body

## Example cURL Request

```bash
curl -X PATCH 'https://wastetrade-api-dev.b13devops.com/listings/admin/123/accept' \
-H 'Authorization: Bearer {jwt_token}' \
-H 'Content-Type: application/json' \
-d '{
    "rejectionReason": "Documentation incomplete"
}'
```
