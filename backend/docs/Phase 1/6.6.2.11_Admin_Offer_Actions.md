# Admin Offer Actions

Admin-only endpoint to perform actions on offers (accept/reject/request information).
Endpoint: `PATCH /offers/admin/{id}/{requestAction}`

## Path Parameters

### id: Offer ID (number)
### requestAction: Action to perform (string)
##### "accept": Approve the offer
##### "reject": Reject the offer
##### "request_information": Request additional information

## Request Body

```json
{
    "rejectionReason": "string (optional, required for reject action, if don't provide, record will have default rejectReason)",
    "message": "string (optional, required for request_information action, if don't provide, record will have default message)"
}
```

```bash

export enum OfferRequestActionEnum {
    ACCEPT = 'accept',
    REJECT = 'reject',
    REQUEST_INFORMATION = 'request_information',
}
```

## Authorization

Requires JWT token with admin or super admin role.

## Actions and Effects

### Accept Action

- Changes offer state to "active"
- Sends acceptance email to buyer (for sell listings) or seller (for wanted listings)
- No additional request body required

### Reject Action

- Changes offer state to "rejected"
- Sends rejection email with provided reason
- Requires rejectionReason, if not have rejectReason will have default reason `"Unsuitable offer"` in request body

### Request Information Action

- Keeps offer state as "pending"
- Sends email requesting additional information
- Requires message, if not have message will have default message `"Please check the offer and provide more information"` in request body

## Example cURL Request

```bash
curl -X PATCH 'https://wastetrade-api-dev.b13devops.com/offers/admin/123/accept' \
-H 'Authorization: Bearer {jwt_token}' \
-H 'Content-Type: application/json' \
-d '{
    "rejectionReason": "Documentation incomplete"
}'
```
